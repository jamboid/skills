Good Needs improvement Poor Marker / dot = measured value within its threshold range
N1Phase-grouped rails

Rails kept as-is, but sorted into three plain-language phases of the load — Painting, Responding, Stability — each with a one-line description, and every acronym spelled out. Lowest-effort narrative; reuses the rail component directly.

N2To-scale load timeline

Paint milestones (FCP, LCP, Speed Index) plotted as flags on a real time axis with the LCP good/needs-work/poor zones shaded — so you literally see how early in the load each one lands. Duration / stability / field metrics that aren't a single moment sit as rails beneath.

N3Stepped "load story"

The strongest narrative: each metric becomes a numbered moment in sequence — "first paint → main content in → visually complete → ready to respond → settled → stays snappy" — with a plain-English caption. Evenly spaced (sequence, not to scale), so it reads like a story rather than a chart.
